# Навчальний проєкт: Інформаційна система бібліотеки

Повностековий веб-додаток для управління бібліотечним каталогом. Проєкт дозволяє переглядати список книг, залишати відгуки, додавати книги до особистого обраного, а адміністраторам — управляти каталогом (додавати/видаляти книги та обкладинки).

## Технологічний стек

* **Бекенд:** PHP 8.2+, Symfony 7, PostgreSQL, Doctrine DBAL / Migrations, LexikJWTAuthentication (авторизація за токенами).
* **Фронтенд:** React 18, TypeScript, Vite, TanStack Router, TanStack Query, Zustand (стейт-менеджмент), TailwindCSS.

## Інструкція з локального запуску

### Вимоги до оточення
Переконайтеся, що на вашому комп'ютері встановлені:
* PHP (>= 8.2) та Composer
* Node.js (>= 18) та npm
* PostgreSQL
* Symfony CLI (опціонально, але рекомендується для запуску сервера)

### Крок 1: Налаштування Бекенда (Symfony)

1. Перейдіть до папки бекенда:
```bash
cd back
```

2. Встановіть PHP-залежності:
```bash
composer install
```

3. Налаштуйте підключення до бази даних. Створіть файл `.env.local` (або відредагуйте `.env`) та вкажіть ваші дані для PostgreSQL:
```env
DATABASE_URL="postgresql://ІМ'Я_КОРИСТУВАЧА:ПАРОЛЬ@127.0.0.1:5432/НАЗВА_БД?serverVersion=16&charset=utf8"
```

4. Згенеруйте ключі для JWT-токенів (необхідно для роботи авторизації):
```bash
php bin/console lexik:jwt:generate-keypair
```

5. Створіть базу даних та застосуйте міграції (міграції автоматично заповнять БД тестовими книгами, жанрами, авторами та користувачами):
```bash
php bin/console doctrine:database:create
php bin/console doctrine:migrations:migrate
```

6. Запустіть локальний сервер бекенда (за замовчуванням на порту 8000 або 8080):
```bash
symfony server:start
```
*(Якщо Symfony CLI не встановлено, можна використовувати вбудований сервер PHP: `php -S localhost:8080 -t public`)*

### Крок 2: Налаштування Фронтенда (React)

1. Відкрийте новий термінал та перейдіть до папки фронтенда:
```bash
cd front
```

2. Встановіть JavaScript-залежності:
```bash
npm install
```

3. Запустіть сервер розробки:
```bash
npm run dev
```

4. Відкрийте у браузері посилання, яке видасть термінал (зазвичай це http://localhost:5173).

## Тестові доступи
База даних вже заповнена тестовими користувачами з різними ролями. Пароль для всіх акаунтів однаковий.

### Адміністратор
Має права на додавання, видалення книг та обкладинок, а також завантаження CSV:
```txt
Email: admin@lib.com
Пароль: 123
```

### Звичайний читач
Може залишати відгуки та додавати книги до обраного:
```txt
Email: client1@lib.com (або client2@lib.com)
Пароль: 123
```

### Управління ролями користувачів (CLI)

У проєкті реалізовано консольну команду для швидкої зміни ролі будь-якого користувача безпосередньо через термінал. Це зручно, якщо вам потрібно надати права адміністратора новому акаунту без прямого втручання в базу даних.

**Синтаксис команди:**
```bash
php bin/console app:change-role <email> <role>
```

**Приклади використання:**

1. Зробити користувача адміністратором:
```bash
php bin/console app:change-role admin@lib.com admin
```

2. Зробити користувача читачем:
```bash
php bin/console app:change-role client1@lib.com user
```
*Примітка: Команду необхідно виконувати, перебуваючи в кореневій директорії бекенду (`cd back`). Система автоматично перевірить існування вказаного email та ролі в базі даних.*
